# Codex Daily Report Skill

这个 skill 用来把本机 Codex 会话记录整理成当天工作日报素材，或把已有日报合并成周报草稿。

默认数据源：

- `~/.codex/state_5.sqlite`
- `~/.codex/sessions/.../*.jsonl`
- 目标工作区及子目录 Git 记录
- 可选的 token 统计命令
- 目标工作区里已经生成的日报 Markdown 文件

## 默认输入

- 日期：用户未指定时，使用当前本地日期。
- 工作区：用户未指定时，优先使用当前工作区。
- 工时：用户未指定时，默认 `8h`。
- 输出：用户指定文件时写入指定文件，否则优先沿用当前工作区已有日报目录；没有约定时写到 `./daily/{date}.md`。

## 工作流

1. 确认日期、工作区、总工时和输出位置。
2. 扫描 `state_5.sqlite` 中目标日期更新的线程。
3. 同时扫描目标日期前后各一天的 `~/.codex/sessions/**.jsonl`，按消息 `timestamp` 判断是否落在目标日期。
4. 只保留 `cwd` 位于目标工作区或其子目录的会话。
5. 清理上下文噪音、重复消息和日报生成线程。
6. 对跨天会话回看必要上下文，确认当天继续推进的真实目标。
7. 扫描目标工作区 Git 提交、未提交改动和 diff 概览，用于补漏和校正工作主线。
8. 可选运行 token 统计命令，在日报开头展示总消耗。
9. 按真实工作主线归并，不按会话数量逐条罗列。
10. 生成简短、可读、面向团队协作的中文 Markdown。

## 写作要求

- 先写工作主线，再写过程细节。
- 每条说明问题或目标、当天动作、当前结果。
- 技术细节只保留能解释交付质量的部分。
- 不把工具调试、路径查找、临时配置排障抬成主要工作，除非它本身就是当天目标。
- 工时总和必须等于用户指定的总工时。
- 如果用于公开分享，落盘前检查会话片段、文件路径、提交信息和截图名称是否需要脱敏。

## 推荐格式

```md
# 工作内容

今日 AI 协作量：
总消耗：1.23 亿 token
计划工时：8h

**工作项｜1、PROJ-123 报表筛选体验优化｜4.5h**
梳理用户反馈中的筛选混乱问题，完成交互边界确认、前端调整和本地验证，当前已形成可评审版本。
**工作项｜2、BUG-456 导出文件名异常修复｜2.0h**
定位特殊字符导致导出失败的问题，补充文件名清洗逻辑和回归验证，当前等待合入主分支。
```
